Node.js / Express
Node app.js

Точка входу Express-додатку. Тут ініціалізується сервер, підключаються middleware, роути та слухається порт.

const express = require('express');
const app = express();

app.use(express.json());
app.use('/api', require('./routes'));

app.listen(3000, () => console.log('Server running'));
  • Завжди виносити роути в окремі файли
  • express.json() — парсинг тіла запиту
  • Порт краще брати з process.env.PORT
Node server.js

Альтернативна точка входу — іноді відокремлюють логіку сервера від логіки додатку (app.js). Зручно для тестування.

const app = require('./app');
const PORT = process.env.PORT || 3000;
app.listen(PORT);
  • При тестуванні app.js не запускає сервер сам по собі
  • Такий поділ — хороша практика для Jest/Supertest

Node controllers/userController.js

Контролер — обробляє запит, викликає сервіс/модель, повертає відповідь. Не містить бізнес-логіки напряму.

exports.getUser = async (req, res) => {
  try {
    const user = await UserService.findById(req.params.id);
    res.json(user);
  } catch (err) {
    res.status(500).json({ error: err.message });
  }
};

Node middleware/auth.js

Middleware для перевірки JWT-токену. Додається перед захищеними роутами.

const jwt = require('jsonwebtoken');

module.exports = (req, res, next) => {
  const token = req.headers.authorization?.split(' ')[1];
  if (!token) return res.status(401).json({ error: 'No token' });
  try {
    req.user = jwt.verify(token, process.env.JWT_SECRET);
    next();
  } catch {
    res.status(403).json({ error: 'Invalid token' });
  }
};
  • Bearer-схема: Authorization: Bearer <token>
  • Секрет зберігати тільки в .env

React hooks/useFetch.js

Кастомний хук для отримання даних з API. Інкапсулює стан loading/error/data.

import { useState, useEffect } from 'react';

export function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}
React context/AuthContext.jsx

Context API — глобальний стан без Redux. Підходить для auth, теми, мови.

import { createContext, useContext, useState } from 'react';

const AuthContext = createContext(null);

export function AuthProvider({ children }) {
  const [user, setUser] = useState(null);
  return (
    <AuthContext.Provider value={{ user, setUser }}>
      {children}
    </AuthContext.Provider>
  );
}

export const useAuth = () => useContext(AuthContext);

JavaScript
JS Що таке замикання (closure)?

Відповідь: Функція, яка «запам'ятовує» лексичне оточення, в якому вона була створена, навіть після того, як зовнішня функція завершила виконання.

function counter() {
  let count = 0;
  return () => ++count;
}
const inc = counter();
inc(); // 1
inc(); // 2
⚠ Нюанси
  • Класична пастка з var в циклі — всі колбеки посилаються на одну змінну
  • let/const вирішують проблему (блочна область)
  • Замикання можуть спричинити витік пам'яті, якщо зберігати великі об'єкти
JS Як працює Event Loop?

Відповідь: JS однопоточний. Event Loop постійно перевіряє Call Stack — якщо він порожній, бере задачі з черги.

  • Call Stack — синхронний код
  • Microtask queue — Promise.then, queueMicrotask (виконується ПЕРЕД макротасками)
  • Macrotask queue — setTimeout, setInterval, I/O
console.log('1');
setTimeout(() => console.log('2'), 0);
Promise.resolve().then(() => console.log('3'));
console.log('4');
// Порядок: 1, 4, 3, 2
⚠ Нюанси
  • Мікротаски виконуються всі до наступного макротаску
  • await — синтаксичний цукор над Promise, теж мікротаск
JS Прототипне наслідування

Відповідь: Кожен об'єкт має [[Prototype]] — посилання на інший об'єкт. При пошуку властивості JS піднімається по ланцюжку прототипів.

const animal = { speak() { return 'sound'; } };
const dog = Object.create(animal);
dog.speak(); // 'sound' — з прототипу
⚠ Нюанси
  • class — синтаксичний цукор, під капотом той самий прототип
  • hasOwnProperty перевіряє тільки власні, не прототипні
JS Що таке this?

Відповідь: this — контекст виконання. Залежить від того, ЯК викликана функція.

  • Метод об'єкта → this = об'єкт
  • Звичайна функція → this = undefined (strict) або window
  • Arrow function → this з лексичного оточення
  • call/apply/bind → явне задання
⚠ Нюанси
  • Arrow functions не мають свого this — часта причина багів у класах/колбеках
  • bind повертає нову функцію, не виконує її
React
React useEffect — як і коли використовувати?

Відповідь: Хук для побічних ефектів — fetch, підписки, DOM-маніпуляції. Виконується після рендеру.

useEffect(() => {
  // виконується після рендеру
  fetchData();
  return () => { /* cleanup */ };
}, [dependency]); // [] — лише при mount
⚠ Нюанси
  • Порожній масив залежностей [] — ефект тільки при mount/unmount
  • Без масиву — після кожного рендеру (небезпечно)
  • Cleanup обов'язковий для підписок і таймерів
  • React 18 StrictMode викликає ефект двічі в dev
React Virtual DOM і Reconciliation

Відповідь: Virtual DOM — JS-представлення реального DOM. При зміні стану React порівнює (diffing) новий і старий vDOM і оновлює тільки змінені вузли.

⚠ Нюанси
  • key prop критичний для списків — без нього React не може ефективно порівняти елементи
  • Не використовувати index як key при динамічних списках
  • React 18: concurrent rendering — рендер переривається, пріоритезується
React Оптимізація: memo, useMemo, useCallback
  • React.memo — не ре-рендерить компонент якщо props не змінились
  • useMemo — мемоїзує результат обчислення
  • useCallback — мемоїзує функцію (посилання не змінюється)
⚠ Нюанси
  • Не застосовувати скрізь — є накладні витрати
  • Профілювати перед оптимізацією (React DevTools Profiler)
  • useCallback потрібен якщо функцію передаємо в мемоїзований дочірній компонент
Node.js / Backend
Node REST vs GraphQL
  • REST: ресурс-орієнтований, кілька ендпоінтів, over/under-fetching
  • GraphQL: один ендпоінт, клієнт запитує тільки потрібні поля, сильна типізація
⚠ Нюанси
  • REST простіший для кешування (HTTP-рівень)
  • GraphQL N+1 проблема — вирішується через DataLoader
  • REST краще для простих CRUD, GraphQL — для складних клієнтів з різними потребами
Node async/await vs Promise.then

Відповідь: async/await — синтаксичний цукор над Promise. Робить асинхронний код схожим на синхронний.

// Promise
fetch(url).then(r => r.json()).then(data => ...)

// async/await
const r = await fetch(url);
const data = await r.json();
⚠ Нюанси
  • Паралельні запити: Promise.all([...]), а не послідовний await
  • await можна використовувати тільки в async функції
  • Завжди обертати в try/catch
DB Що таке індекс у БД?

Відповідь: Індекс — структура даних (B-Tree), яка прискорює пошук по стовпцю. Аналог змісту у книзі.

⚠ Нюанси
  • Прискорює SELECT, але сповільнює INSERT/UPDATE/DELETE
  • Не індексувати поля з низькою кардинальністю (boolean)
  • Складні індекси: порядок стовпців важливий (leftmost prefix)
  • EXPLAIN/EXPLAIN ANALYZE для аналізу запитів
